
# 毕业设计管理与学生能力画像管理需求文档

## 一、毕业设计管理

毕业设计是场景化教学模式下，学生从"课程学习+实践场景"向"岗位综合胜任力"过渡的终结性能力认证环节。毕业设计应与企业真实岗位需求对接，纳入平台双体系认定（学历认定+能力认定）的统一管理框架。

### （一）毕业设计选题管理

#### 1. 建设定位
解决毕业设计"选题来源企业化、选题过程规范化、选题结果可追溯"的问题，确保毕设方向与平台岗位池、实践场景库形成映射关系，避免毕设与岗位能力培养脱节。

#### 2. 业务逻辑
**选题来源：**
- **场景库选题**：从平台已建设的职业岗位-实践场景库中，由企业或教师选择具体场景作为毕设方向，系统自动关联该场景下的能力点、知识点和任务链。
- **企业真实需求新建**：企业基于真实业务需求发布毕设课题，由平台实施人员或教师进行教学化转换，建立选题项目，配置对应的能力模型和测评标准。

**选题流程：**
- 企业/教师/专家发布选题项目 → 配置选题基础信息（名称、所属岗位、关联场景、能力要求、预估周期、学生人数限制）→ 学生浏览选题并申请 → 教师/企业导师审核分配 → 选题结果锁定。

**选题与岗位关联：**
- 每个选题必须关联平台岗位池中的至少一个职业岗位，选题对应的能力点自动继承岗位能力模型，作为毕设评价的依据。

#### 3. 功能构成
| 功能点 | 说明 |
| --- | --- |
| 选题发布 | 企业、教师、专家发布选题，支持从场景库直接引用或新建 |
| 选题配置 | 配置选题基础信息、关联岗位/场景、能力要求、学生容量、指导教师/企业导师 |
| 选题申请 | 学生查看选题列表、查看选题详情（含能力要求、关联场景）、提交申请 |
| 选题分配 | 教师/企业导师审核学生申请，支持双向选择、系统分配、调剂 |
| 选题锁定 | 确定选题后锁定，生成毕设项目档案，进入全生命周期管理 |

#### 4. 与平台现有体系的关系
- 选题关联岗位 → 继承岗位能力模型 → 毕设评价使用岗位测评规则
- 选题关联场景 → 继承场景任务链结构 → 毕设过程可引用场景任务资源

### （二）毕设档案管理

#### 1. 建设定位
实现毕业设计全生命周期的数字化档案管理，学生可维护毕设过程文档、上传作品成果，教师和企业导师可跟踪指导，平台可记录完整的过程性数据。

#### 2. 业务逻辑
**档案结构：**
- 毕设项目基础信息（选题名称、指导教师、企业导师、关联岗位/场景、起止时间）
- 过程性文档（开题报告、中期检查、过程记录、指导记录）
- 成果性文档（毕设作品、论文/报告、演示材料、源代码/工程文件等）
- 评价记录（各阶段评价结果、整改意见）

**档案维护权限：**
- 学生：上传、更新、删除本人毕设文档（在提交前），查看指导反馈
- 教师/企业导师：查看、批注、退回、要求整改
- 辅导员/管理员：查看、导出、归档

**版本管理：**
- 支持文档版本记录，保留历史修改痕迹，便于追溯评价依据。

#### 3. 功能构成
| 功能点 | 说明 |
| --- | --- |
| 档案初始化 | 选题锁定后自动生成毕设项目档案骨架 |
| 文档上传 | 学生按阶段要求上传对应类型文档，支持多种格式 |
| 过程记录 | 学生填写周进展、问题记录；导师填写指导记录 |
| 导师批注 | 导师在线批注、打回修改、标记阶段完成 |
| 整改管理 | 导师发起整改要求，学生提交整改结果，闭环管理 |
| 档案归档 | 毕设结束后自动归档，形成学生个人永久档案 |

### （三）毕设评价管理

#### 1. 建设定位
建立多主体、多维度、与岗位胜任力标准对接的毕设评价体系，评价结果同时服务于学历认定（毕业资格）和能力认定（岗位胜任力画像）。

#### 2. 业务逻辑
**评价主体：**
- **指导教师**：评价学术规范性、知识运用能力、过程表现
- **企业导师**：评价岗位匹配度、实操能力、业务理解、职业素养
- **评阅教师/答辩组**：评价综合质量、创新性、表达能力

**评价标准：**
- 引用岗位能力模型中的能力点，配置各能力点在毕设评价中的权重
- 素养类评价引用岗位测评规则库（如：口语表达能力-清晰/流畅/优秀）
- 评价等级：A/B/C/D/E，与平台统一评价标准一致

**评价结果应用：**
- 学历认定：毕设评价达到规定等级（如C及以上）计入毕业学分
- 能力认定：毕设涉及的能力点测评结果计入学生岗位胜任力画像
- 优秀毕设可标记为"标杆案例"，进入场景库作为后续教学资源

#### 3. 功能构成
| 功能点 | 说明 |
| --- | --- |
| 评价标准配置 | 按选题关联岗位，配置评价维度、能力点权重、等级标准 |
| 过程性评价 | 开题、中期、指导过程中的阶段性评价 |
| 终结性评价 | 成果评阅、答辩评价、综合等级评定 |
| 多主体评价汇总 | 自动汇总教师、企业导师、答辩组评分，按权重计算综合等级 |
| 评价结果认定 | 评价结果同步至学历认定模块和能力画像模块 |
| 优秀案例入库 | 优秀毕设申请入库，经审核后进入平台场景/案例资源 |

### （四）毕业查询管理

#### 1. 建设定位
为学生提供毕业资格和毕设结果的透明化查询通道，支持整改意见的及时反馈和闭环处理。

#### 2. 业务逻辑
**查询内容：**
- 毕业资格审查结果（学分完成情况、课程/场景达标情况、毕设评价等级）
- 毕设各阶段评价结果和综合评定
- 整改意见及整改期限
- 学历认定结果和能力认定结果

**查询权限：**
- 学生：查询本人毕业状态和毕设结果
- 教师/辅导员：查询所指导学生情况
- 管理员：全局查询、统计、导出

#### 3. 功能构成
| 功能点 | 说明 |
| --- | --- |
| 毕业状态看板 | 学生个人毕业资格进度可视化（学分、场景达标、毕设状态） |
| 毕设结果查询 | 查询各阶段评价、综合等级、评语 |
| 整改意见查看 | 查看具体整改要求、期限、整改提交入口 |
| 结果申诉 | 学生对结果有异议可发起申诉，进入复核流程 |
| 毕业证明预览 | 预览学历认定和能力认定结果（徽章/证书预览） |

## 二、学生能力画像管理

学生能力画像是平台双体系培养逻辑（学历体系+能力体系）的集中呈现载体，基于学生在平台的全量学习行为、测评结果、档案材料，通过系统自动统计分析和人工特殊维护相结合的方式生成。

### （一）学生档案管理

#### 1. 建设定位
构建学生自主维护、辅导员审核、结果同步画像的档案管理机制，解决学生过程性成果（证书、竞赛、活动）的数字化沉淀和学分转换问题，同时支持辅导员对学生负面信息的记录管理。

#### 2. 业务逻辑
**正向档案（学生自主上传）：**
- 学生上传：荣誉证书、社会活动佐证材料、获奖证书、技能竞赛成果、实习证明等
- 提交辅导员审核，审核通过后同步至学生画像
- 支持配置学分转换规则：部分证书/竞赛成果可按规则转换为素质拓展学分或能力认定加分

**负向档案（辅导员记录）：**
- 辅导员可记录学生违法违纪、处分等信息
- 配置是否在学生画像中展示（如仅后台管理端可见，或学生本人可见）
- 配置学分扣减规则：与学历体系挂钩，自动扣减对应学分或标记能力认定限制

**档案审核流程：**
- 学生提交 → 辅导员审核（通过/驳回/要求补充材料）→ 审核通过计入画像 → 触发学分转换计算

#### 3. 功能构成
| 功能点 | 说明 |
| --- | --- |
| 材料上传 | 学生上传图片、PDF等佐证材料，填写材料类型、获得时间、颁发机构 |
| 学分转换配置 | 管理员预设证书/竞赛等级与学分转换规则（如国家级一等奖=2学分） |
| 自动转换计算 | 审核通过后，系统自动按规则计算可转换学分，同步至学历体系 |
| 辅导员审核 | 查看材料、审核操作、填写审核意见 |
| 负向信息记录 | 辅导员记录违纪信息、配置展示范围、配置学分扣减 |
| 档案版本记录 | 保留上传历史和审核记录，支持追溯 |

---

### （二）学生能力画像管理

#### 1. 建设定位
实现学生能力画像的自动化生成、可视化展示和可控化维护，为教师提供学生群体能力分析工具，为学生提供自我认知和就业推荐依据，为企业招聘提供人才筛选数据支撑。

#### 2. 业务逻辑
**画像生成逻辑：**
- **系统自动统计**：基于课程任务完成情况、实践场景测评结果、毕设评价、档案材料等全量数据，自动计算各能力点得分、岗位综合评级、画像排名
- **数据更新频次**：教师/管理员可配置画像数据更新周期（实时/每日/每周）
- **学生查询频次**：配置学生端画像查看次数限制或开放时间
- **手动维护**：针对基础信息错误、特殊情况（如数据缺失、申诉修正），教师/管理员可手动调整画像数据，保留调整记录

**画像与就业服务关联：**
- 学生画像与岗位胜任力标准进行AI匹配，自动推荐就业岗位方向
- 企业招聘时，可按岗位能力模型筛选匹配学生

#### 3. 功能构成
| 功能点 | 说明 |
| --- | --- |
| 画像生成引擎 | 自动聚合课程、场景、毕设、档案数据，计算能力点得分和综合评级 |
| 更新频次配置 | 教师配置画像自动更新周期 |
| 查询频次配置 | 配置学生端查询次数、时间段限制 |
| 手动维护 | 教师/管理员对特定学生画像数据进行修正，需填写调整原因 |
| 画像对比分析 | 班级/专业/年级学生能力画像横向对比 |
| 就业推荐配置 | 配置画像与岗位池的匹配推荐规则 |

---

### （三）能力画像展示内容标准

学生能力画像面向学生、教师、辅导员、企业（授权后）展示，分为三大信息板块：

#### 1. 学生基础档案信息
| 信息类别 | 具体内容 |
| --- | --- |
| 身份信息 | 学号、姓名、性别、年级、班级、专业 |
| 形象信息 | 学生上传的个人形象照（可选） |
| 培养信息 | 岗位方向、所属课程列表、已关联实践场景列表 |
| 统计信息 | 已完成课程数、已完成场景数、获得学分总数、档案材料数 |

#### 2. 学生学历评价信息
| 信息类别 | 具体内容 |
| --- | --- |
| 课程学分 | 各课程任务获得的学分及总学分 |
| 课程评级 | 各课程任务的完成等级（A/B/C/D） |
| 期末/结课成绩 | 平台记录或导入的期末考试成绩 |
| 学历认定结果 | 是否达到毕业学分要求、出勤率达标情况 |
| 学历徽章 | 学历体系达标后颁发的认定标识 |

#### 3. 学生能力评价信息
| 信息类别 | 具体内容 |
| --- | --- |
| 岗位能力胜任力画像 | 按岗位能力模型展示各能力领域（行业认知、专业知识、专业技能、通用能力、职业素养）的得分和等级 |
| 岗位综合评级 | 学生在各关联岗位下的综合胜任力等级（A/B/C/D/E） |
| 画像排名 | 在班级/专业/年级中的能力排名（可按岗位方向筛选） |
| 学历+能力综合评定 | 双体系均达标后颁发的综合徽章/证书 |
| 推荐就业岗位方向 | 基于画像与岗位池的AI匹配结果，推荐适配度最高的岗位方向及匹配度百分比 |
