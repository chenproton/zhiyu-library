## 第1章 产品概述

### 1.1 产品定位

能力测评认证平台是一套面向高等院校与职业教育机构的综合性能力评估与认证管理系统。平台覆盖从题库建设、组卷考试、岗位能力认定、毕业设计管理到学生能力画像的全链路业务，为院校提供"教、学、测、评、认"一体化的数字化解决方案。

平台以测评资源的生产与消费为核心主线：教研管理员与教师共同建设高质量的题库与试卷资源，经过标准化审批流程后发布；教师基于已发布的测评资源组织在线考试，系统自动收集考试数据并生成学生能力报告；平台同时支撑岗位能力认定规则的灵活配置、毕业设计全流程跟踪，以及基于多维度数据的学生综合能力画像生成，最终形成"资源生产—考试组织—能力认定—画像呈现"的完整业务闭环。

### 1.2 目标用户

| 角色 | 核心职责 | 典型使用场景 |
|------|---------|-------------|
| 教研管理员 | 生产测评资源、管理审批流程与批次分组 | 创建与维护题库和试卷、配置审批规则、管理资源批次分组、处理资源发布审批 |
| 教师 | 消费测评资源并组织考试、共建教学资源 | 创建在线考试、进行考试评分、查看学生能力报告、参与题库试卷共建、指导毕业设计 |
| 学生 | 参与考试、申请毕业选题、查询个人能力画像 | 参加在线考试、浏览并申请毕业设计选题、提交毕设文档、查看个人能力画像与岗位推荐 |

### 1.3 核心工作流

#### 工作流一：测评资源生产与发布

测评资源（题库、试卷）从创建到发布，需经过标准化生产与审批流程：

1. **创建资源**：教研管理员或教师创建新题库或新试卷，填写名称、简介、封面等基础信息，资源初始状态为草稿。
2. **内容建设**：在题库中添加题目（支持单选、多选、判断、填空、简答、问答六种题型），或在试卷中从题库抽取题目进行组卷，并设置每题分值。
3. **提交审批**：资源建设完成后，提交至审批中心进入审批流程。
4. **审批处理**：审批人查看资源详情，执行通过或驳回操作；通过后可填写审批备注。
5. **发布上线**：审批通过的资源进入待发布状态，发布人确认后资源正式上架，供考试组织等环节引用。

#### 工作流二：考试组织与结果查看

教师基于已发布的测评资源组织在线考试：

1. **创建考试**：教师选择已发布的试卷，创建在线考试实例，填写考试名称、简介、考试须知。
2. **配置对象**：设置考试面向对象（学生或教师），选择参考班级或参考人员。
3. **配置时间**：设置考试开放时间，支持随时开放、定期开放或手动开放三种模式。
4. **发布考试**：确认配置后发布考试，学生可在门户端或考试大厅参与。
5. **结果查看**：考试结束后，教师查看考试结果统计（平均分、最高分、最低分、及格人数等）及学生成绩明细。

#### 工作流三：岗位能力认定

教研管理员配置岗位能力认定规则，系统自动生成学生认定结果：

1. **配置规则**：选择目标岗位，配置该岗位关联的能力域与能力点，设置各能力点权重。
2. **关联任务**：将能力点与场景任务、考试、课程等测评形式关联，设置任务权重。
3. **等级映射**：配置能力掌握度等级区间（如初级、中级、高级、精通等），设置岗位毕业标准档次。
4. **发布规则**：规则配置完成后提交审批并发布，系统开始自动收集关联任务的学生表现数据。
5. **查看结果**：教师与教研管理员查看学生岗位能力达标率、胜任度评分与认定等级。

#### 工作流四：毕业设计管理

覆盖毕业设计从选题到评价的全流程：

1. **发布选题**：教师或管理员发布毕业设计选题，配置关联岗位、指导教师、企业导师、容量与起止时间。
2. **学生申请**：学生在选题列表中浏览已发布选题，提交选题申请并填写申请理由。
3. **档案跟踪**：选题锁定后自动生成毕设档案，学生按阶段（开题、中期、过程、结题）提交文档材料，教师跟踪审核。
4. **多维评价**：指导教师、企业导师、答辩组分别进行评分，系统自动计算综合等级。
5. **结果认定**：评价完成后，将结果同步至学历认定与学生能力画像模块，优秀毕设可标记为标杆案例。

#### 工作流五：学生画像生成与展示

平台聚合多源数据，自动生成并展示学生综合能力画像：

1. **数据采集**：系统自动汇聚课程成绩、考试结果、毕设评价、能力认证、荣誉档案等多维度数据。
2. **画像生成**：基于配置的计算规则与权重，生成学生在五大能力领域（行业认知、专业知识、专业技能、通用能力、职业素质）的得分与等级。
3. **排名计算**：计算学生在班级、专业、年级中的排名情况。
4. **能力展示**：学生查看个人能力画像总览、成长趋势、学历徽章与双证徽章。
5. **岗位推荐**：基于能力匹配度算法，向学生推荐适合的就业岗位。

## 第3章 全局交互规范

### 3.1 页面布局框架

#### 3.1.1 管理平台标准布局

平台管理端采用经典的"顶部导航 + 左侧边栏 + 主体内容区"三栏布局，确保用户在多模块间高效切换。

**顶部固定导航栏**
- 高度固定，始终悬浮于页面最上方，不随内容滚动而消失。
- 左侧区域：展示品牌标识与平台名称，点击后返回平台首页。
- 中部区域：放置顶部快捷导航项（如门户首页、我的服务台、应用服务中心），当前所在项以底部高亮线条标识。
- 右侧区域：依次展示学院筛选下拉框（用于切换当前管理的学院范围）、实时时钟（显示当前日期时间与星期，每分钟刷新）、用户菜单入口（头像 + 用户名 + 角色标签 + 下拉箭头）。

**左侧固定侧边栏**
- 宽度固定，左侧紧贴浏览器边缘，与顶部导航栏共同构成平台的操作框架。
- 顶部区域：放置返回按钮、平台图标与平台名称标签，便于用户识别当前所在系统。
- 菜单区域：按业务域分组展示全部功能模块，支持父级菜单的展开与折叠；当前页面所在的菜单项以高亮背景标识，便于用户定位。
- 在门户首页等特殊页面下，侧边栏自动隐藏，以提供沉浸式的内容浏览体验。

**主体内容区**
- 占据页面右侧剩余空间，背景采用浅灰色，与白色内容卡片形成层次区分。
- 内容区内边距统一，页面内部以白色卡片承载表格、表单、统计等信息，确保视觉聚焦。

#### 3.1.2 预览模式布局

当页面进入预览模式时，平台外壳（顶部导航栏与左侧边栏）完全隐藏，仅保留纯白背景的主体内容区域。该模式主要用于试卷预览、题库预览等需要向外展示或专注查看内容的场景，消除平台操作元素的干扰。

#### 3.1.3 门户首页布局

前台门户首页采用无侧边栏的独立布局：
- 页面背景为纯白色，内容区最大宽度限制并居中展示，保证在大屏下的阅读舒适性。
- 顶部保留品牌栏与主视觉横幅区域，下方按区块模块化排列内容（如快捷导航、测评资源、证书超市、能力认证、毕业设计公告等）。
- 整体风格偏向信息展示与快捷入口，弱化管理操作感，增强学生端的浏览与使用体验。

### 3.2 通用状态流转

题库与试卷两类核心测评资源共用同一套状态流转规则，确保资源从创建到发布的全生命周期管理具备一致的用户认知与操作体验。

#### 3.2.1 状态流转图（文字描述）

资源创建后的初始状态为**草稿**（或**未提交**）。在此状态下，资源仅对创建人及共建人可见，尚未进入任何审批或发布流程。

当资源内容建设完成后，创建人可执行**提交审批**操作，资源状态流转至**审批中**。此时资源被锁定编辑，进入审批人的审核队列。

处于审批中的资源，审批人可执行**通过**操作，资源状态流转至**审批通过**；或执行**驳回**操作，资源状态流转至**已驳回**。与此同时，若资源提交人需要修改内容，可执行**撤回审批**操作，资源状态回退至草稿。

审批通过的资源，进入**待发布**状态，等待最终发布确认。在待发布状态下，执行**发布**操作后，资源正式上架，状态变为**已发布**。已发布的资源可供考试组织、学生练习等业务场景引用。

已发布的资源若需调整，可执行**取消发布**操作，状态回退至待发布。待发布状态下的资源亦可执行**编辑**操作，修改内容后回退至草稿重新走审批流程。

已驳回的资源，创建人可执行**编辑**操作修正问题，修改后状态回退至草稿；修正完成后可再次提交审批。

在任意状态下，用户均可对资源执行**克隆**操作，复制生成一份全新的资源副本，副本初始状态为草稿，与原资源互不影响。

除已发布状态外，其余状态下的资源均可执行**删除**操作，从系统中永久移除。已发布的资源因可能被下游业务引用，不支持直接删除，需先取消发布。

#### 3.2.2 各状态下可执行的操作列表

| 当前状态 | 可执行操作 | 操作说明 |
|---------|-----------|---------|
| **草稿 / 未提交** | 编辑 | 打开资源编辑界面，修改名称、简介、封面、共建人、所属批次等基础信息 |
| | 删除 | 弹出确认对话框，确认后永久删除该资源（注：系统内置的草稿库不可删除） |
| | 提交审批 | 弹出确认对话框，确认后资源进入审批中状态，锁定编辑权限 |
| | 克隆 | 复制生成一份新资源，新资源状态为草稿，内容与原资源一致 |
| **审批中** | 撤回审批 | 弹出确认对话框，确认后资源退回草稿状态，可继续编辑 |
| | 通过（审批人） | 弹出备注输入对话框，确认后资源进入审批通过状态 |
| | 驳回（审批人） | 弹出备注输入对话框，确认后资源进入已驳回状态 |
| | 克隆 | 复制生成一份新资源，新资源状态为草稿 |
| **审批通过** | 进入待发布 | 审批通过的资源流转至待发布状态，等待最终发布确认 |
| | 克隆 | 复制生成一份新资源，新资源状态为草稿 |
| **待发布** | 发布 | 弹出确认对话框，确认后资源正式上架，状态变为已发布 |
| | 编辑 | 打开编辑界面修改内容，修改后状态回退至草稿，需重新提交审批 |
| | 克隆 | 复制生成一份新资源，新资源状态为草稿 |
| | 删除 | 弹出确认对话框，确认后永久删除该资源 |
| **已发布** | 取消发布 | 弹出确认对话框，确认后资源下架，状态回退至待发布 |
| | 克隆 | 复制生成一份新资源，新资源状态为草稿 |
| **已驳回** | 编辑 | 打开编辑界面修正问题，修改后状态回退至草稿 |
| | 提交审批 | 修正后再次提交，进入审批中状态 |
| | 删除 | 弹出确认对话框，确认后永久删除该资源 |
| | 克隆 | 复制生成一份新资源，新资源状态为草稿 |

> **交互细节补充**：
> - 所有涉及状态变更的操作（提交审批、通过、驳回、撤回、发布、取消发布、删除）均需弹出二次确认对话框，防止误操作。
> - 删除、驳回、取消发布等具有破坏性的操作，确认按钮采用醒目的警示色调（红色），与常规操作形成视觉区分。
> - 状态变更成功后，页面顶部弹出轻量提示（Toast），告知用户操作结果，同时列表页对应行的状态徽章实时更新。
> - 审批通过/驳回操作支持填写备注信息，备注内容将在审批详情中展示，便于后续追溯。
## 第4章 通用测评资源管理

## 4.1 题库管理

### 功能概述

题库管理是平台的资源管理中心，面向教研管理员和教师，提供题库资源的查看、筛选、管理与维护能力。用户可在「我的题库」「共建题库」「公共题库」三个维度间切换浏览，支持列表视图和批次分组视图两种展示模式，满足不同场景下的资源管理需求。

### 页面布局

页面整体采用卡片式布局，从上到下依次为：

1. **顶部标题卡片**
   - 左侧：页面主标题「题库管理」+ 副标题「维护题库信息、题目等测评资源管理功能」
   - 右侧操作按钮组：配置审批流程、配置批次分组、导入题库、新建题库

2. **统计面板**
   - 仅在「我的题库」「共建题库」标签下显示
   - 横向排列 5 个统计卡片：题库总数、草稿、审批中、待发布、已发布
   - 各卡片展示对应状态的题库数量，便于用户快速掌握资源分布

## 8. 学生画像管理

**功能概述**

画像列表基于课程任务、实践场景、毕业设计评价、档案材料等全量数据，自动汇聚生成学生综合能力画像。支持按专业、班级进行筛选，提供画像生成、手动调整、对比维度配置等管理操作。

**页面布局**

采用左右分栏布局：
- **左侧专业-班级导航**：按专业分组展示各班级及人数，支持搜索专业或班级；顶部设「全部班级」快捷入口。

- **顶部操作按钮区**：
  - 「生成画像」：手动触发画像生成引擎，重新计算所有学生能力画像。

- **搜索与筛选**：支持按姓名、学号、班级或岗位关键词搜索；按能力等级（A-优秀 / B-良好 / C-中等 / D-及格 / E-不及格）筛选。

## 9. 审批中心

### 9.1 功能概述

审批中心面向具备审批权限的用户（如教研管理员、院系负责人），集中处理题库、试卷、在线考试等测评资源的审批申请。审批人可查看待审批资源详情，执行通过或驳回操作，并追溯历史审批记录，确保测评资源在发布前经过合规审核。

  - **待审批**：展示所有等待审批的资源申请，标签右上角显示待审批数量角标。
  - **已审批**：展示历史已通过或已驳回的审批记录，支持按结果状态二次筛选。

### 9.3 审批记录列表

**列表字段**

| 字段 | 说明 |
|------|------|

| 资源编码 | 资源的唯一编码标识 |
| 版本 | 当前提交审批的版本号 |
| 所属岗位 | 资源关联的岗位方向 |
| 所属批次分组 | 资源归属的建设批次 |
| 创建人 | 提交该资源的原始创建人 |
| 提交审批日期 | 申请提交审批的时间 |
| 状态 | 待审批 / 已通过 / 已驳回，以状态徽章区分 |
| 操作 | 行级操作入口 |

| 按钮 | 显示条件 | 功能说明 |
|------|----------|----------|
| 查看 | 始终显示 | 跳转至资源预览页面，查看完整内容与配置 |
| 通过 | 待审批状态 | 打开通过确认弹窗，审批通过后资源进入待发布或已发布状态 |
| 驳回 | 待审批状态 | 打开驳回确认弹窗，填写驳回原因后退回给创建人修改 |

### 9.4 弹窗说明

**通过确认弹窗**

- **审批意见**：多行文本框，非必填，默认填充「审批通过」。审批人可补充具体意见或建议。
- **底部按钮**：取消 / 确认通过。
- 确认通过后，审批记录状态更新为「已通过」，关联资源按业务规则流转至下一状态。

**驳回确认弹窗**

- **驳回原因**：多行文本框，必填。需清晰说明驳回理由，便于创建人针对性修改。
- **底部按钮**：取消 / 确认驳回。
- 确认驳回后，审批记录状态更新为「已驳回」，资源退回至创建人，创建人可编辑后重新提交审批。

## 11.1 状态定义

### 通用状态（适用于题库、试卷）

| 状态名称 | 说明 |
|---------|------|
| 草稿 | 资源处于初始创建阶段，可自由编辑、删除、克隆 |
| 未提交 | 资源已编辑完成但尚未提交审批，可继续修改或提交 |
| 审批中 | 资源已提交审批流程，等待审批人审核；此状态下可撤回审批或删除 |
| 已驳回 | 审批未通过，已退回至创建人；可根据驳回原因修改后重新提交 |
| 待发布 | 审批已通过，等待管理员手动发布至前台；此状态下可执行发布操作 |
| 已发布 | 资源已正式发布，前台用户可见；已发布资源可取消发布或克隆 |

### 岗位规则状态

| 状态名称 | 说明 |
|---------|------|
| 草稿 | 岗位能力认定规则处于初始配置阶段 |
| 未提交 | 规则配置完成，尚未进入审批流程 |
| 审批中 | 规则已提交，正在等待审批 |
| 已驳回 | 规则审批未通过，已退回修改 |
| 待发布 | 规则审批通过，等待发布 |
| 已发布 | 规则已生效，参与学生的能力认定计算 |
| 无规则 | 该岗位尚未配置任何能力认定规则 |

### 毕业设计选题状态

| 状态名称 | 说明 |
|---------|------|
| 草稿 | 选题处于创建编辑阶段，仅创建人可见 |
| 待审核 | 选题已提交，等待管理员或院系审核 |
| 已发布 | 选题审核通过，已在前台展示，学生可申请 |
| 已锁定 | 选题申请已满或已到截止时间，不再接受新申请 |

## 11.2 类型定义

### 题目类型

| 类型名称 | 说明 |
|---------|------|
| 单选题 | 每题提供多个选项，仅有一个正确答案 |
| 多选题 | 每题提供多个选项，有一个或多个正确答案 |
| 判断题 | 每题提供"正确"或"错误"两种判断选项 |
| 填空题 | 每题提供一个或多个空位，需填写正确答案 |
| 问答题 | 每题需以较长文字进行开放性回答或论述 |
| 简答题 | 每题需以简洁文字回答核心要点 |

### 难度等级

| 等级名称 | 说明 |
|---------|------|
| 简单 | 考查基础概念与直接应用，适合基础巩固 |
| 中等 | 考查综合理解与分析能力，需要一定推理 |
| 困难 | 考查深度掌握与灵活运用，涉及复杂场景或多知识点交叉 |

### 档案材料类型

| 类型名称 | 说明 |
|---------|------|
| 荣誉证书 | 学生获得的各类荣誉、奖项、表彰证明 |
| 竞赛成果 | 学生参与学科竞赛、技术竞赛的成果记录 |
| 社会活动 | 学生参与社会实践、志愿服务、社团活动的证明材料 |
| 实习证明 | 学生在企业或机构实习的经历与评价证明 |
| 技能证书 | 学生获得的专业技能、职业资格、语言能力等证书 |
